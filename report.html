Overview

We will be creating an ASP generated maze that is visualized using Javascript. First we will develop the rules and constraints for the solvable mazes in our proposed Greek mythology universe. Next we will develop a front-end interface to visualize the generated maze (a reach goal will be to make it playable). Our maze will be different than the one in the PCG book in that we hope to further expand their example maze to create different levels of difficulty to allow the user to play different mazes. Rather than just having one requirement to solve the maze we hope to make it multi-stepped with constraints also on the maze configuration.

We propose to develop three different mazes using answer set programming. Once we have our logic completed we will be using HTML, CSS, and Javascript to create a website displaying the mazes. For the constraints of our mazes we plan on making each maze a different mythological story. We will be creating our mazes so that each maze is solvable, complex, and requires a minimum of one task be completed before the user can exit the maze.

Design and Technical Approach

ASP

We will be using clingo as our solver to execute our ASP logic. We will make three levels, one easy, one medium, and one hard. The logic and constraints will be slightly different for each.

Easy: The player is trapped in a labyrinth with the deadly minotaur. The player must get the sword, slay the minotaur, and exit the labyrinth. The size of the maze will be 10x10. The player will always spawn at (1,1) and the exit will always spawn at (10,10). The sword and the minotaur will both be accessible. You will only be able to confront the minotaur when you have the sword, if you try before that you will lose.

Medium: The player is Hermes, messenger of the gods. Of course, Hermes needs his winged sandals to travel, which he has lost! The player must find their winged sandals and then use them to traverse over “sky” tiles to retrieve an important message and then reach the exit. The size of the maze will be 15x15. The player will always spawn at (1,1) and the exit will always spawn at (15,15). The player will have to use the sandals to reach the exit but may or may not need them to get the message. The player cannot finish the maze without the message. If the player tries to go over a sky tile without their sandals, they lose.

Hard: You are an adventurer who has been trapped in the underworld and is trying to escape. Hades has caught wind of this and has donned his cap of invisibility to try to stop you. You must retrieve the magical key to the underworld and then use it to unlock the exit and then proceed to the exit without getting caught by Hades who will be invisible to you. The size of the maze will be 15x15. The player will always spawn at (1,1) and the exit will always spawn at (15,15). If the player runs into Hades or tries to escape the underworld before unlocking the exit, they will lose.

Visualization

For the visualization of the mazes we plan on exporting the output we receive from clingo and importing it as a text file into our Website which will be written in HTML, CSS, and Javascript code. To toggle between the different mazes we will be having a dropdown button or checkboxes so that the user can choose which level they want to view. Based on that user input we plan on displaying the corresponding maze. For the Javascript portion of this project we plan on parsing through the text file and assign colors to the divs on the page which will serve as our canvas. The overall themes of our website will be based on Greek Mythology and the obstacles and tasks that are generated in our mazes will reflect Greek Mythology.

As an additional bonus we hope to make our mazes playable by adding javascript event handlers. Based on the position of the cursor it will determine the users place on the maze. We also hope to add a timer into our webpage that will start once the user is hovering over the start of the maze.

Documentation and Access

We will be sharing code through a GitHub repository. There will be detailed readme file with what our project is and how the implementation works. In the implementation of our project we plan on each making commits and detailed comments of the code so that each member knows what changes have been made.

Evaluation

To determine whether our mazes are solvable, complex, and logically correct we will be using our Javascript visualization to verify that the mazes perform like our english logic. From the visualization we will be able to see whether a task object is being blocked by walls or whether the end of the maze is reachable by the user or not. The visualization will also serve as a way to compare the difficulty of the levels. To test whether our mazes are difficult enough we will be comparing the time it takes to generate the mazes.
